# Отчёт: последние шаги боевой системы (4 этапа)

Краткий инженерный конспект четырёх последовательных шагов по боевой экономике и UX. Доменная логика (стоимость, КД, `AbilityResourceGate`, `applyAbilityCost`, тик раунда) на этапах 1–3 не пересматривалась в этом документе; этап 4 — только UI.

---

## 1. Фундамент resource layer (мана / энергия / КД по раундам)

**Цель:** общий слой ресурсов для всех юнитов, без переписывания всего combat pipeline.

**Суть:**
- В `CombatUnitState` хранятся `Mana`, `MaxMana`, `Energy`, `MaxEnergy`, `AbilityCooldowns map[AbilityID]int` (осталось полных раундов).
- Реестр способностей: `CostMana`, `CostEnergy`, `CooldownRounds`; базовая атака бесплатна (`AbilityIsFree`).
- `AbilityResourceGate` в `ListValidTargets` / валидации; `applyAbilityCost` в `ResolveAbility` после успешной валидации.
- `tickRoundResources` в `AdvanceTurn` при завершении полного раунда: КД −1, лёгкая регенерация маны/энергии.
- UI/AI для выбора способностей: `AvailableAbilities` / `SpecialAbilitiesUsable` (где уместно).

**Критерий:** `go test ./...` проходит; бой не ломается.

---

## 2. Recovery: отдых на R без лечения ОЗ

**Цель:** убрать бесплатное восстановление HP от «простого отдыха» в мире; лечение — только явные источники.

**Суть:**
- `ApplyWorldRest` больше не увеличивает `CurrentHP` по отряду; сохранён только аварийный выход из soft-lock (лидер 1 ОЗ, если никто не может сражаться).
- `advanceWorldTurn` по R без изменений контракта.
- POI (`healExploreFraction`, костёр и т.д.) и бой лечения не трогались.

**Критерий:** R не лечит; POI/прочие источники работают; тесты зелёные.

---

## 3. Vertical slice способностей (играбельная демонстрация экономики)

**Цель:** 3–5 способностей с реальными мана/энергия/КД, чтобы экономика была видна в игре.

**Суть в реестре (пример):**
- Базовый удар — без ресурса.
- Дальний удар — энергия.
- Лечение / масс-лечение — мана и КД (массовое — с более заметным КД).
- Усиление — мана + энергия + КД.
- Панель способностей показывает **все** спец. способности; выбор клавиатурой по полному `actor.Abilities()`.

**Критерий:** игрок видит траты и КД в бою; AI использует `AvailableAbilities`.

---

## 4. Player-facing UI pass (этот шаг)

**Цель:** сделать мана/энергию/стоимость/КД и причину недоступности читаемыми без перегруза HUD и без изменения доменной логики.

**Изменения (представление):**

| Место | Что видит игрок |
|--------|------------------|
| HUD v1, строка фазы | `Мана a/b · Энергия c/d` (активный союзник, ход игрока). |
| HUD v2, блок «Ваш ход» | Текст ресурсов + две микрополосы (мана / энергия). |
| Панель способностей (v1 и v2) | Строка стоимости: `мана N · энергия M · КД Nр · ещё Nр` при активном ожидании КД. |
| Недоступная способность | Вторая строка: `КД: ещё N р.` / `Мана: нужно …, сейчас …` / `Энергия: …`; рамка 2px цветом причины (КД / мана / энергия). |
| Панель «ХОД» (v1) | Способность с расшифровкой стоимости; отдельная строка ресурсов; превью урона/лечения — пятая строка, если панель высока. |
| Summary v2 | К выбранной способности добавляется строка стоимости. |

**Хелперы в `internal/battle`** (только текст/формат, не правила):
- `AbilityCostLinePlayerRU` — стоимость для подписи.
- `AbilityUnavailableHintRU` — короткая причина по коду `ValidationResult` (те же gate).
- `ActorResourceLineRU` — слова «Мана»/«Энергия» и текущие/макс.

**Тема:** `ResourceManaFill`, `ResourceEnergyFill`, `AbilityBlockCooldownBrd`, `AbilityBlockManaBrd`, `AbilityBlockEnergyBrd`; `DrawResourceBarMicro`.

**Не ломалось:** контракты лейаута и hit-test (только высота блока активного юнита v2 под микробары); поток боя, inspect, доменные проверки.

**Техдолг:** отдельные иконки ресурсов; tooltips; анимации полос; локализация бeyond RU-строк.

**Критерий:** `go test ./...` проходит.
