# Отчёт: UX ясности боевого опыта, награды лидеру и знаков обучения

## 1. Изменённые места UI

| Место | Изменение |
|-------|-----------|
| **Post-battle, шаг «Победа»** | Несколько коротких строк сводки: кто получил боевой опыт, пояснение про усиление обычной атаки, павшие/резерв при наличии, знаки обучения; подсказка перехода к награде лидеру. |
| **Post-battle, шаг награды** | Две строки: отдельно от боевого опыта отряда + «Выберите вариант». |
| **Inspect (карточка бойца)** | Человекочитаемые формулировки по опыту, бонусу к обычной атаке, лечению; блок «Профиль» без сырого `unit_id`; пояснения для резерва и 0 ОЗ; повышение с именами шаблонов. |
| **Состав (F5)** | Одна строка про боевой опыт и резерв. |
| **HUD** | Строка знаков обучения в том же духе, что и post-battle/inspect. |

## 2. Формулировки

- **Боевой опыт** — явно отделён от **награды лидеру** (post-battle + шаг выбора).
- **Знаки обучения** — как награда за победу и трата на повышение в лагере.
- **Опыт → сила обычной атаки** — коротко в сводке и в карточке (шаги `CombatXPStepsPerBasicAttackBonus` без вывода формулы в HUD).

## 3. Что не менялось в механике

- `ApplyVictoryCombatXPForActiveSurvivors`, константы опыта и знаков.
- Reward pool и `ApplyReward` для лидера.
- Порядок: sync → XP → результат боя → знаки → post-battle.

## 4. Код

- `internal/progression/victory_summary.go` — `BuildVictoryProgressionSummary`, `HeroShortLabel`.
- `internal/postbattle/postbattle.go` — `Flow.VictorySummaryLines`, `Begin(outcome, lines)`.

- `internal/ui/postbattle.go` — layout с учётом числа строк сводки; отрисовка списка.
- `internal/game/update.go` — вызов сводки при победе.
- `internal/ui/character_inspect.go`, `formation_overlay.go`, `panels.go` — тексты.

## 5. Тесты

- `internal/progression/victory_summary_test.go` — выжившие, павший, резерв, короткое имя героя.

## 6. Ограничения

- Длинные имена в сводке при большом отряде сворачиваются в «N участников» (уже в логике сводки при >4 имён в одной строке — для двух всегда полные имена).
- Полное переносы длинных строк в post-battle не делались — одна строка = один `drawSingleLineInRect` (возможен клип на узком окне).

## 7. Следующие шаги (3–5)

1. При необходимости — перенос/сокращение длинных строк post-battle по ширине панели.
2. Локализовать единообразно подписи наград лидера, если появятся новые типы.
3. Краткий баннер в explore после первой победы (one-shot tutorial) — опционально.
4. Отображение «до следующего бонуса от опыта» без введения «уровней» — опционально.
5. Синхронизация терминов с будущим журналом квестов, если появится.
