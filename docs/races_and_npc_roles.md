## 1. Назначение документа

- Это **game design doc** по **расам / фракциям**, **боевым ролям** и **юнитным линиям** (NPC и отряды игрока как единая логика шаблонов).
- Документ задуман как **источник правды** для дальнейшей корректировки дизайна и как база под реализацию: шаблоны юнитов, разнообразие рекрута, ветки прогрессии, inspect UI, мир.
- **Пока подробно проработана только одна фракция — Империя** (см. [§5](#5-империя)). Остальные слоты в [§4](#4-фракции--расы) и [§6](#6-будущие-фракции) — заготовки под расширение.
- Документ **не привязан жёстко к коду**: имена полей ниже — **design-schema**, а не обязательные идентификаторы репозитория.

## 2. Общие принципы

- **Юнит** в смысле дизайна = **фракция / раса** + **линия развития** + **tier** + **роль / архетип** (конкретная запись — `unit_id`).
- **Позиция на поле** не запрещает действия жёстко: любой юнит может стоять в любой клетке; **эффективность** зависит от роли, типа атаки и тактики, а не от жёсткого «только задний ряд».
- **Базовые типы действия** (в терминах боя):
  - **ближний бой** (`melee`);
  - **дальний бой** (`ranged`);
  - **лечение** (`heal`).
- **Базовые правила по умолчанию** (для типичного применения обычной атаки / лечения):
  - **melee** — по умолчанию бьёт только **передний ряд врага**;
  - **ranged** — по умолчанию может бить **любую вражескую цель**;
  - **heal** — по умолчанию может применяться к **любому союзнику** (для групповых способностей — уточнение в `combat_rule` конкретного юнита).
- **Роль важнее «позиции по умолчанию»**: сетка не заменяет чтение архетипа и `combat_rule`.
- Документ отражает **ранний этап дизайна**, а не финальный баланс; числа в `stat_profile` — **качественные** теги (low / medium / high и т.д.), не финальные числа кода.

### Согласованные на текущий момент правила (кратко)

| Правило | Суть |
|--------|------|
| Melee default | Обычная атака ближнего боя — в типичном случае только **передний ряд врага**. |
| Ranged default | Обычная атака дальнего боя — **любая цель** среди врагов. |
| Heal default | Лечение — **любой союзник**; групповые способности описываются отдельно. |
| Position | **Нет жёсткого запрета** по клетке; эффективность от роли и типа атаки. |

## 3. Общая модель данных юнита (design-schema)

Набор полей, который пригодится для шаблонов, таблиц рекрута и UI. **Это не код.**

| Поле | Назначение |
|------|------------|
| **faction_id** | Идентификатор фракции/расы (например `empire`). |
| **line_id** | Линия развития внутри фракции (`warrior`, `archer`, `healer`, …). |
| **unit_id** | Уникальный id юнита-шаблона. |
| **display_name** | Имя для UI. |
| **tier** | Ступень 1…5. |
| **archetype** | Ролевой архетип (см. таблицы ролей фракции). |
| **attack_type** | `melee` / `ranged` / `heal`. |
| **stat_profile** | Качественный профиль (hp, damage, defense, initiative, …). |
| **base_ability** | Название базовой активной способности (дизайн). |
| **combat_rule** | Ключевое правило применения в бою (текст или ключ для данных). |
| **upgrade_to** | Список следующих `unit_id` (пусто = конец ветки). |
| **early_game_availability** | Уровень доступности в ранней игре (см. таблицы фракции). |

Дополнительно в черновике Империи использовалось поле **line** (синоним **line_id** в тексте линий).

## 5. Империя

### 5.1 Общая идея фракции

- **Империя** — организованная **человеческая / имперская** фракция с выстроенной военной и сопутствующей доктриной.
- **Три основные линии развития юнитов:**
  1. **воинская** (`warrior`);
  2. **стрелковая** (`archer`);
  3. **линия хилов** (`healer`).
- **Развилка на две специализации** начинается с **tier 2 → tier 3** (после общего tier 1 и общего tier 2 в линии).

### Дерево развития (текстовая схема)

```
Воинская:     Новобранец → Оруженосец →┬→ Танк:     Щитоносец → Латник → Гвардеец
                                        └→ ДД:       Мечник → Чемпион → Каратель

Стрелковая:   Рекрут-лучник → Стрелок →┬→ Чистый DD: Лучник → Меткий стрелок → Снайпер
                                        └→ Яд:       Отравитель → Токсикант → Чумной стрелок

Хилы:         Послушник → Аколит →┬→ Одиночный: Целитель → Хранитель жизни → Высший целитель
                                 └→ Массовый: Пастырь → Хранитель хора → Архиерей милости
```

### 5.4 Тактические правила Империи (ключи данных)

| Ключ | Содержание |
|------|------------|
| **melee_default_rule** | Обычная атака может бить только **вражеский передний ряд**. |
| **ranged_default_rule** | Обычная атака может бить **любую вражескую цель**. |
| **heal_default_rule** | Лечение может применяться к **любому союзнику**; групповые способности уточняются в `combat_rule` юнита. |
| **position_rule** | Любой юнит может стоять в **любой клетке**; эффективность от **роли** и **типа атаки**, не от жёсткого запрета по клетке. |

### 5.6 Практический вывод для реализации

- **Раннеигровой пул** (ориентир): новобранец, рекрут-лучник (и варианты через ранний recruit: оруженосец, стрелок, послушник, аколит с оговорками по редкости).
- **Recruit sources на старте** логично наполнять из **стартового доступа** и **раннего recruit**; **редкий ранний** — веса, лимиты, кулдауны.
- **Не выдавать рано** (без специальных сценариев): tier 4–5 и помеченные **недоступен рано** — чтобы не ломать кривую силы.

## 7. Статус документа

- Документ **дизайн-ориентированный**; не каждое поле уже отражено в коде.
- **Числа** в статах — **качественные** (low / medium / high и т.д.), не финальные числовые балансы.
- **Империя** на текущем этапе — **единственная** полностью расписанная фракция; остальное — задел.
- Правки по балансу и именам — **сюда**, затем при необходимости синхронизировать узкие черновики (`empire_roles_and_units.md`).
