# Recruit / new member acquisition — foundation (отчёт)

## 1. Executive summary

- **Цель:** канонический способ **получать новых членов партии** как `hero.Hero`, по умолчанию в **`Reserve`**, без ломки battle pipeline и без мета-overhaul.
- **Реализация:** `hero.NewRecruitHero()` — базовый шаблон стата; `party.AddToReserve`, лимит **`MaxPartyMembers = 12`**; в explore **F9** — демо-acquisition (новобранец в резерв) + баннер; подсказка в нижней панели обновлена.
- **Совместимость:** `PlayerCombatSeeds` только из **Active**; пост-бой, recovery, F5 formation без изменения правил; новый герой участвует в **ApplyWorldRest** как и остальные в резерве.

## 2. Состояние до изменений

- Состав можно было переставлять Active/Reserve, но **расширять ростер** было возможно только правкой кода (`TwoMemberDemoParty` и т.д.).
- Не хватало единого **доменного** API «добавить участника» и игровой точки входа.

## 3. Целевая модель этапа

| Элемент | Правило |
|--------|---------|
| Новый участник | Тот же тип **`hero.Hero`**, что и у остальных. |
| Куда попадает | Только **`Party.Reserve`** через `AddToReserve`. |
| Active | Не меняется при рекруте; игрок переводит через **F5**. |
| Лимит | **`MaxPartyMembers`** (12): Active + Reserve. |
| Старты статов | `NewRecruitHero`: 9 HP, 2 атака и т.д. (шаблон этапа). |
| Точка входа | **F9** в explore — временный демо-поток до world/recruit sources. |
| Имя в UI | `Новобранец N`, N = порядковый номер в резерве после добавления. |

## 4. Изменённые файлы

| Файл | Содержание |
|------|------------|
| `internal/hero/recruit.go` | **Новый:** `NewRecruitHero()`, `RecruitDisplayName(n)`. |

## 5. Инварианты

- Бой по-прежнему только **Active** + `CanEnterBattle()`.
- `Party` — единственный источник истины по составу.
- Рекрут не создаёт параллельную сущность «recruit unit».

## 6. Упрощения / временные решения

- Нет экономики, квестов, мира, пост-боя как источника рекрута.
- F9 — **явно этапный** демо-ввод (см. подсказку внизу экрана).
- Имена «Новобранец N» привязаны к **порядку в резерве**, не к глобальному ID сессии.

## 7. Ручная проверка

- [ ] Explore → F9 → баннер, в F5 новый ряд в резерве.
- [ ] Перевод в Active, бой, HP sync.
- [ ] R — HP у резерва тоже растёт.
- [ ] Набить 12 участников до отказа — сообщение «Отряд полон».
- [ ] Регрессии: F5, R, бой.
