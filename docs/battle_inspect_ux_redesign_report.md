# Отчёт: UX-redesign RMB-inspect (карточка в бою)

## 1. Что было неудобно раньше

- Карточка строилась как **единый список строк** из `buildInspectLines` (союзник) и `buildEnemyInspectLines` (враг): длинные предложения про опыт, лечение и повышение шли подряд с одинаковым визуальным весом.
- **Слабая иерархия**: заголовок и ОЗ не выделялись; секции «профиль / статы / способности» смешивались с пояснениями.
- Визуально карточка напоминала **debug-список полей**, а не тактическую панель.

## 2. Новый layout

- **Верхняя золотая полоса** + **цветная вертикальная полоса** слева (союзник / враг по `Theme.AllyAccent` / `Theme.EnemyAccent`).
- **Крупный заголовок** — имя (тот же источник, что и раньше: `inspectPrimaryTitle` / `CombatUnit.Name()`).
- **Бейджи** — одна строка: ранг · роль · тип атаки (из шаблона или упрощённо для legacy-врага).
- **Контекст** — «В бою · слот» или «Противник».
- **ОЗ** — крупная строка + **полоска HP** (`DrawHPBarMicro`), цвет текста ОЗ: норма / низкий запас / опасность.
- **Секции с подзаголовком и линией** (`DrawThinAccentLine`):
  - **Профиль** — 1–2 короткие строки (линия + примечание из шаблона при наличии).
  - **Показатели** — одна строка «Атака · Защита · Инициатива»; для союзника дополнительно строка про лечение.
  - **Способности** — список с маркером «·», по одному имени на строку.
  - **Развитие** — только союзник: компактная сводка опыта/удара + до 6 строк из существующего `inspectPromotionLines` (те же данные и правила, что в составе).

- **Footer** — короткая подсказка по закрытию и переключению.

Ширина панели слегка уменьшена (480 px max), чтобы карточка воспринималась компактнее.

## 3. Блоки карточки

| Блок | Содержание |
|------|------------|
| Заголовок | Имя |
| Бейдж | Ранг, роль, тип атаки |
| Контекст | Бой / противник |
| ОЗ | Числа + бар |
| Профиль | Название линии, фракция/заметка (по данным шаблона) |
| Показатели | Боевые числа + лечение (союзник) |
| Способности | Имена способностей |
| Развитие | Опыт и ветки повышения (союзник) |
| Подвал | Esc / ПКМ |

## 4. Ally vs enemy

- **Общее**: тот же каркас секций, те же примитивы отрисовки, те же цвета темы.
- **Союзник**: блок **Развитие** (опыт + строки promotion через `inspectPromotionLines`), строка про лечение, профиль из `hero`/`unitdata`.
- **Враг**: без развития и без строки лечения; профиль и бейдж из шаблона или legacy `Def`; **сырой `TemplateUnitID` в текст карточки не выводится** (при отсутствии шаблона — нейтральная формулировка без id).

## 5. Что сознательно не делалось

- Новый глобальный UI framework, анимации, отдельное окно на весь проект.
- Изменение боевой логики, progression, economy.
- Перерисовка battle HUD вне карточки inspect.
- Отдельный шрифт крупного кегля (иерархия за счёт отступов, высот строк, цвета ОЗ и секций).

## 6. Изменённые файлы

- `internal/ui/battle_inspect.go` — модель `battleInspectCardModel`, билдеры ally/enemy, отрисовка секций, `flattenBattleInspectCardText`.
- `internal/ui/battle_inspect_test.go` — тесты профиля врага, отсутствия утечки секретного id, наличия блока развития у союзника.
- `docs/battle_inspect_ux_redesign_report.md` — этот отчёт.

## 7. Логичные следующие UX-шаги

1. Локальная подсветка строки при наведении в списке способностей (если появится hover в overlay).
2. Иконки-заглушки роли (vector) рядом с бейджем без подключения арта.
3. Скрытый режим отладки (например, по зажатому Shift) с показом внутренних id — только при явном флаге.
4. Слияние визуального языка с карточкой состава (F5) для полного единообразия (общий helper секций).
