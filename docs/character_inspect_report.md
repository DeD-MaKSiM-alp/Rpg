# Character inspect / sheet UI — отчёт этапа

## 1. Executive summary

- **Цель:** дать игроку канонический просмотр **конкретного** участника партии (роль, строй/резерв, ОЗ, статы, боевой опыт, бонусы, способности) без отдельного metagame overhaul.
- **Реализация:** overlay **«карточка бойца»** поверх уже существующего экрана **состава (F5)**; открытие **I**, закрытие **I / Esc / F5**; при открытой карточке **↑↓** переключают выбранного бойца, **Enter / ← / →** отключены, чтобы не путать с перестановкой.
- **Данные:** только **`hero.Hero`** и **`party.Party.HeroAtGlobalIndex`**; для новобранцев — поле **`RecruitLabel`** (заполняется при найме F9 / world-recruit).
- **Совместимость:** explore, бой, резерв, progression без изменения правил.

## 2. Состояние до изменений

- Состав (formation) показывал строки с ролью, слотом и HP, но **без** детальных статов, XP и расшифровки бонусов.
- У `Hero` не было подписи для новобранца в данных (только сообщение в баннере при найме).

## 3. Целевая модель этапа

| Элемент | Решение |
|--------|---------|
| Точка входа | Только **formation (F5)** → **I**. |
| Закрытие | **I**, **Esc**, **F5** (первый раз закрывает карточку; из списка состава F5 по-прежнему выходит в explore при закрытой карточке). |
| Имя | `RecruitLabel` или `MemberRoleCaption` / `ReserveRowCaption`. |
| Обязательное | Положение (строй/резерв), готовность к бою, ОЗ, атака/защита/инициатива, бонус базовой атаки (итог + разложение), боевой опыт, лечение (2+бонус), список способностей (RU). |

## 5. Инварианты

- Карточка читает **канонический** `Hero`, не снимок боя.
- Active и reserve обрабатываются **одинаково** по индексу списка formation.
- Inspect **не** меняет reorder/reserve сам по себе; при открытой карточке управление составом заблокировано.

## 6. Упрощения / follow-up

- Нет портретов, инвентаря, вкладок, сравнения героев, inspect из боя.
- Способности — краткие русские имена в UI, не полный tooltip по бою.

## 7. Ручная проверка

- [ ] F5 → I → карточка лидера → Esc → состав.
- [ ] Карточка открыта → ↑↓ другие члены → данные меняются.
- [ ] Резерв: карточка показывает резерв и «вне боя».
- [ ] Новобранец с `RecruitLabel` после F9/world recruit — имя в заголовке.
- [ ] 0 HP в строю — строка «не может сражаться».
- [ ] После боёв с ростом XP — числа на карточке обновляются.
- [ ] Enter/стрелки влево-вправо при открытой карточке **не** двигают строй.

## 8. Follow-up

- Портреты, отдельные вкладки, сравнение двух героев.
- Экипировка, таланты, журнал.
