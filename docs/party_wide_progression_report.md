# Party-wide progression — foundation (инженерный отчёт)

## 1. Executive summary

- **Цель этапа:** убрать однополюсную «лидер-центричную» прогрессию после боёв и ввести **канонический** слой роста для **нескольких активных бойцов**, не ломая `Battle → Post-battle → Party` и выбор награды лидеру.
- **Реализация:** у `hero.Hero` поле **`CombatExperience`**; после **победы** выжившие участники **активного строя** (есть в бою) получают **+1** опыта (`progression.ApplyVictoryCombatXPForActiveSurvivors`); в `CombatUnitSeed` бонус к базовой атаке: **`BasicAttackBonus + CombatExperience/4`** (константа `CombatXPStepsPerBasicAttackBonus`).
- **Лидер:** по-прежнему **только лидер** получает **выбор награды** (`ApplyReward`); это не дублирует party-wide XP (слой разделён в комментариях и UI).
- **Совместимость:** `PlayerCombatSeeds` / `CombatUnitSeed` → следующий бой; резерв, recruit, recovery, HP без изменения правил; поражение/отступление **не** дают боевого опыта.

## 2. Состояние до изменений

- **Прогрессия в `Hero`:** статы, способности, `HealPower`, `BasicAttackBonus` и т.д.
- **Награды после победы:** `postbattle` → `GenerateRewardOffer` / `ApplyReward` только к **`party.Leader()`** (Active[0]).
- **Путь:** `BattleOutcomeVictory` → `syncPartyFromBattle` → post-battle (результат → награда лидеру).
- **Ограничение:** остальные активные бойцы не получали канонического роста после победы; reserve не участвует в бою — естественно не рос.

## 3. Целевая модель этапа

| Элемент | Правило |
|--------|---------|
| Хранение | `Hero.CombatExperience` (int), старт 0. |
| Кто получает | **Только** юниты игрока в **конце победного** боя: `TeamPlayer`, валидный `PartyActiveIndex`, **`IsAlive()`**. |
| Кто не получает | **Поражение, отступление**; резерв; **павший** активный участник; активные с 0 HP до боя (не попали в сиды — нет юнита). |
| Размер прироста | `CombatExperiencePerVictorySurvivor = 1` за победу. |
| Эффект в бою | `EffectiveBasicAttackBonusForCombat()` = `BasicAttackBonus + CombatExperience/4` (целочисленно). |
| Награда лидеру | Без изменений по смыслу; применяется к лидеру после выбора опции. |
| Recruit | Начинает с 0; после вывода в active и побед растёт как остальные. |

**Осознанный design choice:** party-wide слой даёт **базовый** рост урона базовой атаки; **лидер** по-прежнему получает **крупные** выборы наград (статы, способности). Это не дублирует одно и то же: XP — «пассивный опыт партии», reward — «фокус на лидере».

## 5. Инварианты после изменений

- Прогрессия party-wide хранится **в `Hero`**, не в побочной структуре.
- **Battle** строится из снимка `Hero` через **`CombatUnitSeed`** (учёт XP в бонусе базовой атаки).
- **Reserve** не получает XP без участия в бою (текущее правило этапа).
- **Лидерские награды** и **CombatExperience** разделены по смыслу и коду.
- Порядок при победе: **sync HP → XP →** мир/post-battle **→** награда лидеру на следующем шаге.

## 6. Что упрощено / временно

- Нет уровней, таблиц XP, ролей, веток, тренировки в резерве.
- Рост только через **базовую атаку** (чтобы не раздувать модель полей).
- Нет отображения числа опыта в formation (только подсказка в post-battle на шаге результата).

## 7. Ручная проверка (checklist)

- [ ] Один герой: победа → подсказка про опыт → награда лидеру → следующий бой сильнее по базовой атаке после накопления XP.
- [ ] Несколько active: победа, все живы → у всех +1 `CombatExperience` (косвенно — следующий бой).
- [ ] Один пал в бою, победа остальных: павший **не** получает XP (нет живого юнита).
- [ ] Резерв вне боя: XP **не** растёт.
- [ ] Recruit → в active → победа → растёт как остальные.
- [ ] Поражение / отступление: без прироста XP.
- [ ] Регрессии: F5, R, бой, post-battle награда.
