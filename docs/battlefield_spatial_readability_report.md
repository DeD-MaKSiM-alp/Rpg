# Отчёт: spatial readability поля боя (formation 2×3, v2)

## 1. Executive summary

- **Цель этапа:** сделать поле боя **пространственно читаемым**: структура **2 ряда × 3 ячейки** на сторону, явные **передний / задний** ряд относительно **центра столкновения**, без необходимости знать `PlayerSlotForPartyIndex` из кода.
- **Результат:** геометрия слотов перестроена: **глубина к центру боя — по оси X** (полосы «задний / передний»), внутри полосы **три слота по вертикали (Y)**. Добавлены **подписи** «Задний» / «Передний», **подложки полос** и **слабая** обводка пустых ячеек; передний ряд дополнительно отмечен **вертикальным акцентом** со стороны линии столкновения.
- **Совместимость:** порядок **`BattlefieldSlotCells`** и логика слотов **`BattleRow` + index 0..2** не менялись; **hit-test** по-прежнему опирается на **`BattlefieldTokens`** и **`UnitRects`**; runtime боя, HUD ростеров и feedback не трогались.

---

## 2. Состояние до изменений

- **`computeBattlefieldPlacements`** раскладывал ряды **по оси Y** (нижняя полоса = front, верхняя = back), а **три слота ряда — по X**. Визуально «перед» читался как **низ экрана**, а не как **близость к вертикальной линии между командами**.
- Ячейки сетки рисовались **одинаковой** обводкой `PanelTitleSep`, пустые слоты **конкурировали** с токенами по весу.
- Не было **подписей рядов** и **группирующих** подложек: шесть прямоугольников на сторону сливались с фоном.
- **Смысл терялся** именно в несоответствии ожидания «фронт к центру боя» и фактической геометрии «фронт = низ».

---

## 3. Целевая модель этапа

| Понятие | Как читается на экране |
|--------|-------------------------|
| **Линия столкновения** | Вертикальная линия по центру зоны battlefield (между половинами игрока и врага). |
| **Передний ряд** | Полоса **ближе к этой линии** (у игрока — правее в его половине; у врага — левее в своей). |
| **Задний ряд** | Полоса **у внешнего края** своей половины (у игрока — левее; у врага — правее). |
| **3 ячейки в ряду** | Внутри полосы ряда — **три слота сверху вниз** (индекс 0..2). |

**Согласование с formation:** `PlayerSlotForPartyIndex` по-прежнему задаёт `(row, index)`; порядок **party.Active[0..2] → передний ряд**, **[3..5] → задний**. Экран formation overlay уже описан как «сверху вниз: передний ряд (3), затем задний» — вертикальное расположение слотов **внутри** полосы ряда **лучше согласуется** с этим списком, чем прежняя горизонтальная тройка в ряду.

## 4. Что именно изменено

### `internal/battle/hud_layout.go`

- Добавлены **`BattlefieldRowLabels`**, **`BattlefieldPlayerBack` / `BattlefieldPlayerFront`**, **`BattlefieldEnemyFront` / `BattlefieldEnemyBack`** для подписей и прямоугольников полос.
- Тип **`BattlefieldRowLabel`**, внутренний **`battlefieldV2Geometry`**.
- **`computeBattlefieldPlacements`** переписан: глубина **front/back по X**, слоты **по Y**; подписи «Задний» / «Передний»; защита от переполнения: **`depthW` не превышает** `(halfW - depthGap)/2`, убрана некорректная раздувающая нижнюю сетку минимальная высота `contentH`.

### `internal/ui/battlefield_tokens.go` — `DrawBattlefieldV2Scene`

- Порядок отрисовки: **подложки полос** → **акцент на переднем ряду** (вертикаль у внутренней кромки) → **линия столкновения** → **слабые** границы ячеек → **текстовые подписи** → **токены** → feedback floats (как было).

### `internal/ui/theme.go`

- Цвета **`BattlefieldBackRowBand`**, **`BattlefieldFrontRowBand`**, **`BattlefieldFrontRowBorder`**, **`BattlefieldEmptyCellBorder`** для ненавязчивого различия рядов и ослабления пустых ячеек.

---

## 5. Ключевые инварианты после изменений

- Передний ряд **визуально ближе** к центру столкновения; задний — **к внешнему краю** стороны.
- Порядок **`BattlefieldSlotCells`**: игрок front 0..2, back 0..2, враг front 0..2, back 0..2 — **сохранён**.
- **Hit-test** по **токенам** и карточкам — **без смены контракта**.
- Пустые ячейки **слабее** по обводке; занятые слоты **считываются** через токены и полосы рядов.

---

## 6. Что остаётся упрощённым / временным

- Подписи короткие («Передний» / «Задний») из-за узких полос — при необходимости можно сократить/скейлить шрифт отдельным проходом.
- Фон центра battlefield по-прежнему **placeholder** (`SceneTint` / панели в `battle_panels.go`) — без terrain/ground art.
- Нет отдельной иконографии «колонок» 0/1/2 — только индекс на токене и порядок в ростере.

---

## 7. Ручная проверка (checklist)

- [ ] Бой с **одним** союзником — слот в ожидаемом ряду/ячеек по правилам спавна.
- [ ] Бой с **несколькими** союзниками — **передний/задний** ряд и **три** вертикальных слота читаются без подсказки.
- [ ] **Hover / target** по токенам и карточкам работает как раньше.
- [ ] **Active unit**, **dead unit**, **wait** — обводки/feedback на токенах.
- [ ] Сопоставление с **порядком party.Active** и подписями formation overlay.
- [ ] **HUD** (ростеры, нижняя панель) без регрессий.
- [ ] При необходимости — **v1** layout (Style 0) не использует новую отрисовку v2.
