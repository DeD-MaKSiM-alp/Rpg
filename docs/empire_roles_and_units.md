> **Мастер-документ по расам, ролям и юнитам:** [`races_and_npc_roles.md`](races_and_npc_roles.md)  
> Там же — общие принципы, **design-schema** полей, оглавление фракций и статус документа.  
> Этот файл удобен как **короткая выжимка только по Империи** для быстрых правок; при расхождении **сначала обновляют мастер-док**, затем при необходимости синхронизируют этот черновик.

---

Документ фиксирует каноническую модель **фракции Империя**: линии развития, **tier’ы**, **unit_id**, **тактические правила** и **раннюю доступность** для будущей интеграции в код (templates, recruit tables, inspect, progression, world outcomes).

---

## Общая идея

Пока прорабатывается **только одна раса/фракция: Империя**.

У Империи есть **три основные линии развития**:

1. **воинская**;
2. **стрелковая**;
3. **линия хилов**.

На **2-м уровне (tier 2)** каждая линия **делится на 2 специализации** (ветки progression).

### Принцип позиционирования

**Любого юнита можно поставить на любую позицию** — жёсткого запрета нет.  
Эффективность зависит от **роли**, **типа атаки** и общей тактической целесообразности, а не от жёсткой привязки «только задний ряд».

### Базовые боевые правила (общие)

| Правило | Содержание |
|--------|------------|
| **Ближний бой** | Обычной атакой может бить **только вражеский передний ряд**. |
| **Дальний бой** | Может бить **любую вражескую цель**. |
| **Лечение** | Может применяться к **любому союзнику** (детали по веткам — в `combat_rule` юнита: одиночное / групповое). |

## 2. Иерархия юнитов Империи

### Общие поля записи юнита

| Поле | Назначение |
|------|------------|
| **line** | Линия развития (`warrior` / `archer` / `healer`) |
| **tier** | Ступень (1–5) |
| **unit_id** | Внутренний идентификатор |
| **display_name** | Имя для UI |
| **archetype** | Боевая роль (тип архетипа) |
| **attack_type** | `melee` / `ranged` / `heal` |
| **stat_profile** | Профиль статов (теги уровня: low, medium, high, …) |
| **base_ability** | Базовая активная способность |
| **combat_rule** | Ключевое правило роли в бою |
| **upgrade_to** | Список следующих `unit_id` (пустой = конец ветки) |

## 3. Общие тактические правила Империи (машиночитаемые ключи)

| Ключ | Содержание |
|------|------------|
| **melee_default_rule** | Обычная атака может бить только **вражеский передний ряд**. |
| **ranged_default_rule** | Обычная атака может бить **любую вражескую цель**. |
| **heal_default_rule** | Лечение может применяться к **любому союзнику** (для групповых способностей — уточняется в `combat_rule` юнита). |
| **position_rule** | Любой юнит может стоять в **любой клетке**; эффективность зависит от **роли** и **типа атаки**, а не от жёсткого запрета по клетке. |

## 4. Доступность юнитов Империи в ранней игре

Матрица для **recruit sources**, **мира** и **раннего баланса**.

### Обозначения доступности

| Обозначение | Смысл |
|-------------|--------|
| **Стартовый доступ** | Может быть у игрока **в самом начале**. |
| **Ранний recruit** | Можно получить в **первых** recruit-источниках мира. |
| **Редкий ранний** | Может появиться рано, но **не должен** быть частым. |
| **Недоступен рано** | Для ранней игры **не выдаётся** обычным образом. |

## 5. Что можно вынести в систему данных

| Артефакт | Назначение |
|----------|------------|
| **Unit templates** | Справочник по `unit_id` + поля из раздела 2. |
| **Recruit tables** | Веса и ограничения с учётом раздела **4**. |
| **Inspect labels** | `display_name`, линия, tier, archetype. |
| **Progression branches** | Граф `upgrade_to` по линиям и веткам. |
| **World recruit outcome** | Привязка событий к `unit_id` или пулу с фильтром по **ранней доступности**. |

Текущий кодовый vertical slice (`Hero`, `Party`, бой) может сосуществовать с этим документом: по мере роста **`unit_id`** и шаблоны подключаются к каноническому `Hero` или слою определений.
